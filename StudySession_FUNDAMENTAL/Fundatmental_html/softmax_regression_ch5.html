큰 그림 — 머신러닝 3종 세트

모든 모델은 세 가지로 이루어진다. 가설(Hypothesis)은 예측하는 식, 비용 함수(Cost)는 예측이 얼마나 틀렸는지 재는 자, 옵티마이저(Optimizer)는 비용을 줄여 나가는 방법이다.

이 장의 핵심 통찰은 단순하다. 선형 회귀(3장) → 로지스틱 회귀(4장) → 소프트맥스 회귀(5장)는 이 셋 중 앞의 둘만 갈아 끼운 것이다. 옵티마이저(경사 하강법)와 학습 루프는 끝까지 변하지 않는다.

장문제가설비용 함수
ch3회귀 (연속값)Wx + bMSE
ch4이진 분류sigmoid(Wx + b)BCE
ch5다중 분류softmax(WX + B)Cross Entropy

옵티마이저는 셋 모두 경사 하강법이고, 파이토치 학습 루프도 항상 같은 3박자다.

학습 루프 — ch3·4·5 공통
optimizer.zero_grad()   # 이전 기울기 초기화
cost.backward()         # 비용을 미분해 기울기 계산
optimizer.step()        # W, b 업데이트
핵심

5장 내내 머릿속에 둘 한 줄: “가설과 비용만 바뀐다.” 새 개념처럼 보이는 softmax·크로스 엔트로피도, 사실 4장에서 배운 것을 클래스 개수만큼 확장한 것뿐이다.

5-1

원-핫 인코딩

미리보기

다중 분류의 정답 라벨을 어떻게 적을 것인가의 문제다. 정수로 적으면 클래스 사이에 가짜 거리·순서가 생긴다. 그래서 정답 칸만 1, 나머지는 0인 원-핫 벡터로 표현한다.

정수 인코딩의 함정

제일 단순한 방법은 클래스마다 정수를 매기는 것이다. 예를 들어 Banana=1, Tomato=2, Apple=3. 그런데 여기에 선형 회귀의 MSE(제곱 오차)를 적용하면 문제가 드러난다.

실제값이 Tomato인데 Banana로 예측한 경우와, 실제값이 Apple인데 Banana로 예측한 경우의 제곱 오차를 비교해 보자.

$$ (2 - 1)^2 = 1 \qquad\qquad (3 - 1)^2 = 4 $$

즉 기계에게 “Banana는 Apple보다 Tomato에 더 가깝다”는 엉뚱한 정보를 주는 꼴이 된다. 세 과일은 그냥 서로 다른 범주일 뿐, 순서나 거리 관계가 없는데도 말이다.

원-핫 인코딩 — 모두를 균등하게

그래서 원-핫 인코딩을 쓴다. 클래스 개수만큼 차원을 만들고, 정답 인덱스 칸만 1, 나머지는 0으로 둔다.

원-핫 벡터 — 클래스 3개
강아지 = [1, 0, 0]
고양이 = [0, 1, 0]
냉장고 = [0, 0, 1]

이렇게 하면 어떤 두 클래스를 골라도 제곱 오차가 똑같이 2가 된다 (유클리드 거리 $\sqrt{2}$). 클래스 간 관계가 균등해지는 것이다 — “모두 똑같이 남남”이라는 사실을 올바로 표현한다.

예외 — 순서가 있을 때

클래스에 순서가 있으면 정수 인코딩이 오히려 맞다. {baby, child, adult} 나 {1층, 2층, 3층} 처럼 순서 정보가 의미 있는 경우다. 순서 없는 일반 분류만 원-핫을 쓴다.

이 원-핫 라벨이 5-2에서 소프트맥스 출력 [0.24, 0.72, 0.04](예측)과 정답 [0,1,0](원-핫)을 비교하는 토대가 된다.

5-2

소프트맥스 함수와 비용 함수

미리보기

가설은 H(X) = softmax(WX + B). 모델이 뱉은 점수를 확률로 바꾸고, 비용 함수로 크로스 엔트로피를 써서 정답(원-핫)과의 차이를 잰다. 그리고 이 크로스 엔트로피는 4장의 BCE와 본질적으로 같은 식이다.

① 가설 — 점수를 확률로

붓꽃 문제는 특성 4개(꽃받침·꽃잎의 길이·너비)로 품종 3개(setosa·versicolor·virginica) 중 하나를 맞힌다. 문제가 하나 있다. 입력은 4차원인데 출력은 클래스 개수만큼 3차원이어야 한다. 그래서 가중치 곱으로 4→3 차원을 변환하고, 그 결과를 $z = [z_1, z_2, z_3]$ 라 부른다.

그런데 이 $z$ 는 아직 그냥 점수(실수)다. 합이 1도 아니고 음수도 가능하다. 이걸 확률로 바꾸는 함수가 소프트맥스다.

$$ p_i = \frac{e^{z_i}}{\sum_{j=1}^{k} e^{z_j}} \qquad (i = 1, 2, \dots, k) $$

식은 복잡해 보여도 하는 일은 두 단계다. 지수 $e^x$ 를 씌워 전부 양수로 만들고 → 전체 합으로 나눠 비율로 만든다. 끝이다. 결과는 0~1 사이이면서 합이 1인 확률 벡터가 된다.

② 행렬 연산 — 크기만 맞추면 끝

샘플 여러 개를 한 번에 처리할 때는 차원을 맞추는 게 전부다. 특성 $f$ 개, 클래스 $c$ 개일 때 가설을 행렬로 쓰면 다음과 같다.

$$ \hat{Y} = \text{softmax}(XW + B) $$
  • X : (샘플수 × $f$) — 입력 데이터
  • W : ($f$ × $c$) — 학습 대상. 붓꽃은 4×3
  • B : (샘플수 × $c$) — 편향
  • Ŷ : (샘플수 × $c$) — 각 행이 한 샘플의 확률 벡터

③ 비용 함수 — 크로스 엔트로피

한 샘플의 비용은 다음과 같다.

$$ \text{cost} = -\sum_{j=1}^{k} y_j \log(p_j) $$

여기서 $y_j$ 는 원-핫 라벨(정답 칸만 1), $p_j$ 는 소프트맥스 확률이다. 그런데 $y$ 가 원-핫이라 마법이 일어난다. 정답이 아닌 칸은 $y_j = 0$ 이라 통째로 사라지고, 정답 칸($y_j = 1$) 하나만 살아남는다.

$$ \text{cost} = -\log(p_{\text{정답 칸}}) $$

즉 비용은 언제나 “정답 칸에 분배된 확률의 $-\log$” 다. 정답 칸에 확률을 많이 줬으면($p \to 1$) 비용은 0, 적게 줬으면($p \to 0$) 무한대로 폭발한다.

전체 데이터에 대한 평균까지 씌우면 최종 비용 함수가 된다.

$$ \text{cost}(W) = -\frac{1}{n}\sum_{i=1}^{n}\sum_{j=1}^{k} y_j^{(i)} \log\!\big(p_j^{(i)}\big) $$

④ 크로스 엔트로피 = BCE의 일반화

이 식은 4장에서 배운 이진 크로스 엔트로피(BCE)와 본질적으로 같다. BCE에서 출발해 보자.

$$ \text{cost} = -\big[\, y\log H(X) + (1-y)\log(1-H(X)) \,\big] $$

클래스가 2개이므로 치환만 한다. $y \to y_1$, $\,1-y \to y_2$, $\,H(X) \to p_1$, $\,1-H(X) \to p_2$.

$$ = -\big(y_1\log p_1 + y_2\log p_2\big) = -\sum_{i=1}^{2} y_i \log p_i $$

이제 합의 위끝 $2$ 를 $k$ 로 풀어 주면 소프트맥스의 크로스 엔트로피와 완전히 동일해진다.

$$ = -\sum_{i=1}^{k} y_i \log p_i $$
한 바퀴가 닫힌다

이진 분류는 소프트맥스의 $k=2$ 특수 케이스다. 거꾸로 소프트맥스 CE에 $k=2$ 를 넣으면 BCE가 튀어나온다. 3장→4장→5장이 “가설·비용만 갈아 끼운 같은 구조”라는 그림이 여기서 완성된다.

가설비용 함수
ch3 선형Wx + bMSE
ch4 로지스틱sigmoid(Wx + b)BCE (= CE, k=2)
ch5 소프트맥스softmax(WX + B)CE (k ≥ 2)
5-3

파이토치로 구현하기

미리보기

비용 함수를 로우 레벨(식 그대로) → 점점 한 줄로 줄여 나간다. 신기하게 모든 방식이 같은 값을 낸다. 그리고 모델도 직접 만든 W·b 에서 nn.Linear, 클래스로 추상화된다.

먼저 도구상자 별명부터. 코드 맨 위의 이 한 줄이 핵심이다.

import
import torch
import torch.nn as nn               # 클래스 버전 도구상자 (nn.Linear, nn.CrossEntropyLoss)
import torch.nn.functional as F     # 함수 버전 도구상자 (F.softmax, F.cross_entropy)

F 는 torch.nn.functional 의 별명이다. softmax, log_softmax, cross_entropy 같은 딥러닝 함수가 모여 있는 상자다. nn 은 같은 기능의 클래스 버전이다.

비용 함수를 점점 줄이기

아래 네 가지는 전부 같은 값을 낸다. 전제는 모델 출력 점수 z (3×5) 와 정수 라벨 y = [0, 2, 1] 이다.

① low-level — 식을 손으로
hypothesis = F.softmax(z, dim=1)          # 점수 → 확률 (dim=1: 행마다 합이 1)

y_one_hot = torch.zeros_like(hypothesis)  # 같은 크기의 0 텐서
y_one_hot.scatter_(1, y.unsqueeze(1), 1)  # 정수 라벨 → 원-핫 (정답 칸에 1)

cost = (y_one_hot * -torch.log(hypothesis)).sum(dim=1).mean()

마지막 줄이 식 $-\sum y_j \log p_j$ 의 평균을 그대로 옮긴 것이다. 안에서 밖으로 읽으면: -torch.log(...) 로 확률에 $-\log$ → 원-핫과 칸별 곱으로 정답 칸만 남기고 → .sum(dim=1) 으로 $\sum$ → .mean() 으로 $\frac{1}{n}\sum$.

낯선 두 줄 풀이

dim=1 — “행 방향(가로, 클래스끼리)으로 합이 1 되게 하라”. 각 샘플 안에서 확률을 분배한다는 뜻이다. dim=0 이면 샘플끼리 합쳐져 의미가 망가진다.

scatter_ — y.unsqueeze(1) 로 모양을 (3×1)로 바꾼 뒤, 각 행의 그 위치에 1을 꽂는다. 끝의 _ 는 원본을 직접 덮어쓴다는 표시(in-place)다.

② log + softmax 합치기 → log_softmax
torch.log(F.softmax(z, dim=1))  ==  F.log_softmax(z, dim=1)   # 완전히 동일
③ 나머지까지 합치기 → nll_loss (원-핫 불필요!)
F.nll_loss(F.log_softmax(z, dim=1), y)   # 정수 라벨 y를 바로 받는다

nll_loss (Negative Log Likelihood)는 원-핫이 필요 없다. 정수 라벨에서 “정답 칸만 골라 $-$값, 평균”을 알아서 해 준다.

④ 전부 합치기 → cross_entropy
F.cross_entropy(z, y)   # = log_softmax + nll_loss
①softmax → log → one_hot 곱 → sum → mean식을 직접
↓
②F.log_softmax → one_hot 곱 → sum → meanlog+softmax 결합
↓
③F.nll_loss(F.log_softmax, y)원-핫 사라짐
↓
④F.cross_entropy(z, y)한 줄

모델 4종 — 직접 하던 걸 위임

학습 루프 뼈대는 ch3부터 똑같다. 바뀌는 건 가설·비용을 계산하는 윗부분뿐이다. 데이터는 붓꽃류 x_train(8×4), y_train(클래스 3개)이다.

① low-level — W·b 직접
W = torch.zeros((4, 3), requires_grad=True)   # 4→3 변환, 그래서 4×3
b = torch.zeros((1, 3), requires_grad=True)
optimizer = optim.SGD([W, b], lr=0.1)

for epoch in range(1001):
    hypothesis = F.softmax(x_train.matmul(W) + b, dim=1)
    cost = (y_one_hot * -torch.log(hypothesis)).sum(dim=1).mean()

    optimizer.zero_grad()
    cost.backward()
    optimizer.step()
② high-level — cross_entropy 로 두 줄 교체
z = x_train.matmul(W) + b           # softmax 안 씌움! 날것 점수(logits)
cost = F.cross_entropy(z, y_train)  # 내부에 softmax 포함 + 정수 라벨 그대로
③ nn.Module — nn.Linear 에 위임
model = nn.Linear(4, 3)   # 입력 4 → 출력 3 (출력 = 클래스 개수!)
optimizer = optim.SGD(model.parameters(), lr=0.1)

for epoch in range(1001):
    prediction = model(x_train)               # = XW + b (아직 logits)
    cost = F.cross_entropy(prediction, y_train)
    optimizer.zero_grad(); cost.backward(); optimizer.step()

ch3 선형 회귀 때는 출력이 1이었는데, 이제 출력 = 클래스 개수(3) 인 게 유일한 차이다.

④ class — nn.Module 상속
class SoftmaxClassifierModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.linear = nn.Linear(4, 3)   # Output 3!

    def forward(self, x):
        return self.linear(x)

model = SoftmaxClassifierModel()

③과 기능이 완전히 같다. 모델을 클래스로 감쌌을 뿐이다. 모델이 커지면(층 여러 개) 이 구조가 관리하기 편해서 표준처럼 쓴다.

⚠ 가장 흔한 함정

F.cross_entropy 와 nn.CrossEntropyLoss 는 내부에 softmax가 이미 들어 있다. 그래서 이걸 쓸 땐 가설에서 softmax를 또 씌우면 안 된다. 날것 점수(logits)를 그대로 넘겨야 한다. 두 번 씌우면 확률을 또 뭉개 학습이 망가진다.

또한 라벨은 원-핫이 아니라 정수 그대로 넘긴다. 다음 MNIST에서 Y 를 0~9 정수로 바로 넘기는 이유다.

5-4

MNIST 손글씨 분류

미리보기

지금까지 한 걸 진짜 손글씨 숫자에 적용한다. 이번엔 클래스 10개(0~9). 28×28 픽셀 이미지를 784차원 벡터로 펴서 nn.Linear(784, 10) 으로 분류한다. 학습 루프는 여전히 그 3박자다.

① 데이터 모양

이미지 한 장은 28×28 픽셀이다. 모델은 벡터를 받으므로 일렬로 펴서 784차원 벡터로 만든다. 그래서 X.view(-1, 28*28) — “배치는 그대로, 픽셀은 784로 펴라”. 입력 784, 클래스 10.

② 셋업 — torchvision · DataLoader

데이터 로드
mnist_train = dsets.MNIST(root='MNIST_data/', train=True,
                          transform=transforms.ToTensor(), download=True)

data_loader = DataLoader(mnist_train, batch_size=100,
                         shuffle=True, drop_last=True)

drop_last=True 는 개수가 모자란 마지막 자투리 배치를 버린다 — 그 배치가 경사 하강법에서 상대적으로 과대평가되는 것을 막는다.

③ 모델 · 비용 · 옵티마이저

설계 — 입출력만 784→10
linear = nn.Linear(784, 10, bias=True).to(device)   # 입력 784 → 출력 10
criterion = nn.CrossEntropyLoss().to(device)        # 내부에 softmax 포함!
optimizer = torch.optim.SGD(linear.parameters(), lr=0.1)

5-3에서 쓴 nn.Linear·CrossEntropyLoss 그대로다. 입출력 숫자만 바뀌었다. .to(device) 는 GPU를 쓸 때 그 메모리로 보내는 것이다(CPU면 무시).

④ 학습 루프 — 또 그 3박자

training loop
for epoch in range(training_epochs):
    for X, Y in data_loader:
        X = X.view(-1, 28*28).to(device)   # (100, 784)로 펴기
        Y = Y.to(device)                   # 라벨은 0~9 정수 (원-핫 아님)

        optimizer.zero_grad()
        hypothesis = linear(X)             # logits — softmax 안 씌움
        cost = criterion(hypothesis, Y)
        cost.backward()
        optimizer.step()

Y 가 정수 그대로 인 점에 주목하자 — CrossEntropyLoss 가 알아서 처리한다. 돌리면 비용이 약 0.53 → 0.27 로 떨어지고, 정확도는 약 88% 가 나온다.

⚠ deprecated API — 갈아 끼울 것

테스트 코드의 .test_data / .test_labels 는 deprecated 다. 요즘 torchvision에선 경고가 뜨거나 동작하지 않는다. .data / .targets 로 바꾼다.

수정 — .data / .targets
# 옛날: mnist_test.test_data / mnist_test.test_labels  ← 동작 X
X_test = mnist_test.data.view(-1, 28*28).float().to(device)
Y_test = mnist_test.targets.to(device)

prediction = linear(X_test)
correct = torch.argmax(prediction, 1) == Y_test   # 가장 큰 확률 칸 = 예측한 숫자
accuracy = correct.float().mean()

분류의 최종 판정은 늘 argmax 다. 10개 확률 중 가장 큰 칸의 인덱스가 모델이 고른 숫자이고, 그게 정답과 같으면 맞힌 것이다.

5장 한 바퀴

원-핫(5-1)  →  softmax 가설 + 크로스 엔트로피(5-2)  →  4가지 구현으로 추상화(5-3)  →  MNIST 실전(5-4)
  • 다중 분류 라벨은 원-핫으로 — 정수 인코딩은 가짜 거리를 만든다.
  • 가설 softmax(WX+B) 는 점수를 합이 1인 확률로 바꾼다 (지수 → 합으로 나눔).
  • 비용은 정답 칸 확률의 −log — 크로스 엔트로피는 4장 BCE의 k≥2 일반화다.
  • cross_entropy 에 softmax 내장 — 가설에서 또 씌우지 말 것. 라벨은 정수.
  • MNIST는 784→10, 판정은 argmax, .data/.targets 로 수정.
  • ch3·4·5 전부 가설·비용만 갈아 끼운 같은 3박자 루프.
