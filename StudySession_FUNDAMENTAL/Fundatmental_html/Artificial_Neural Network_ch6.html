CHAPTER 06

인공 신경망
(Artificial Neural Network)

하나의 인공 뉴런인 퍼셉트론에서 출발해, 층을 쌓아 비선형 문제를 푸는 다층 퍼셉트론(MLP), 그 학습을 가능케 하는 역전파, 그리고 실제 데이터를 분류하며 마주치는 과적합과 기울기 소실까지 — 딥러닝의 토대를 한 번에 정리한다.

개념 평가·활성화함수 → 구조 퍼셉트론·MLP → 학습 역전파 → 실습 digits·MNIST → 안정화 규제·정규화
06·01

머신 러닝 기본 개념 정리하기

딥러닝 또한 머신 러닝에 속하므로, 머신 러닝의 특징은 곧 딥러닝의 특징이기도 하다. 본격적인 신경망에 앞서 평가·분류·활성화 함수 같은 토대 개념을 정리한다.

1. 머신 러닝 모델의 평가

데이터를 훈련(Training)·검증(Validation)·테스트(Test) 세 가지로 분리하는 것이 일반적이다. 검증 데이터의 역할이 핵심인데, 검증 데이터는 성능을 평가하기 위한 것이 아니라 성능을 조정하기 위한 것이다. 즉 과적합 여부 판단과 하이퍼파라미터 튜닝에 사용된다.

  • 하이퍼파라미터 — 사람이 직접 정하는 값. 학습률, 은닉층 수, 뉴런 수, 드롭아웃 비율 등.
  • 매개변수 — 기계가 훈련을 통해 스스로 바꾸는 값. 가중치와 편향.
비유하자면 훈련 데이터는 문제지, 검증 데이터는 모의고사, 테스트 데이터는 수능이다. 검증으로 튜닝을 끝낸 모델은 검증 데이터에도 어느 정도 최적화되므로, 진짜 성능은 한 번도 보지 못한 테스트 데이터로 평가한다.

데이터가 부족해 검증·테스트를 나누기 어렵다면 k-폴드 교차 검증을 사용하기도 한다.

2. 분류(Classification)와 회귀(Regression)

  • 이진 분류 — 둘 중 하나를 정하는 문제. (합격/불합격, 정상메일/스팸)
  • 다중 클래스 분류 — 세 개 이상의 정해진 선택지 중 하나를 정하는 문제.
  • 회귀 — 분리된 값이 아닌 연속된 값을 결과로 갖는 문제. (성적·주가 예측)

3. 지도 학습과 비지도 학습

머신 러닝은 지도·비지도·강화 학습으로 나뉜다. 이 책은 주로 지도 학습을 다룬다. 지도 학습은 레이블(정답, 실제값 \(y\))과 함께 학습하며, 기계는 예측값 \(\hat{y}\)과 실제값의 오차를 줄이는 방향으로 학습한다. 비지도 학습은 레이블 없이 학습하며 군집(clustering)·차원 축소가 대표적이다.

4. 샘플(Sample)과 특성(Feature)

독립 변수의 개수가 \(n\)개, 데이터 개수가 \(m\)개인 행렬 \(X\)에서, 하나의 행(데이터)을 샘플, 종속 변수 예측에 쓰이는 각 독립 변수를 특성이라 부른다. (DB에서는 샘플을 레코드라 부른다.)

5. 혼동 행렬(Confusion Matrix)

정확도(맞춘 수 / 전체 수)만으로는 세부 내용을 알 수 없다. 양성/음성 이진 분류에서 결과를 네 가지로 나눈 것이 혼동 행렬이다. 각 열은 예측값, 각 행은 실제값을 나타낸다.

예측: 참(Positive)예측: 거짓(Negative)
실제: 참TP (양성 맞춤)FN (음성이라 했으나 양성)
실제: 거짓FP (양성이라 했으나 음성)TN (음성 맞춤)

True/False는 정답을 맞췄는지, Positive/Negative는 제시한 답이다. 여기서 두 지표가 파생된다.

$$\text{Precision(정밀도)} = \frac{TP}{TP+FP} \qquad \text{Recall(재현률)} = \frac{TP}{TP+FN}$$
  • 정밀도 — 양성이라 대답한 전체 중 실제 양성의 비율.
  • 재현률 — 실제 양성 데이터 중 양성으로 맞춘 비율.

6. 과적합(Overfitting)과 과소적합(Underfitting)

과적합은 훈련 데이터를 과하게 학습해 노이즈까지 암기한 상태다. 훈련 오차는 낮지만 테스트 오차는 높아진다. 검은 강아지만 학습한 모델이 흰 강아지를 강아지가 아니라고 판단하는 식이다. 반대로 과소적합은 훈련이 부족해 훈련 데이터에 대해서도 정확도가 낮은 상태다.

학습 곡선의 신호

에포크(epoch, 전체 훈련 데이터에 대한 훈련 횟수)가 늘면 훈련 오차는 계속 감소하지만, 어느 시점부터 테스트 오차는 다시 증가한다. 테스트 오차가 증가하기 직전에 훈련을 멈추는 것(조기 종료)이 바람직하다.

7. 비선형 활성화 함수(Activation Function)

활성화 함수는 입력을 받아 수학적 변환을 수행해 출력을 만드는 함수다. 핵심 조건은 비선형이어야 한다는 점이다.

활성화 함수가 비선형이어야 하는 이유

선형 함수 \(f(x)=Wx\)로 은닉층을 여러 번 쌓으면 \(y(x)=f(f(f(x)))=W\cdot W\cdot W\cdot x\)가 된다. 그런데 \(W^3=k\)로 두면 \(y(x)=kx\)가 되어 층을 한 번 쌓은 것과 차이가 없다. 즉 선형 함수로는 깊이를 쌓는 의미가 사라진다.

단, 선형층(linear/projection layer)이 무의미한 것은 아니다. 학습 가능한 가중치가 생긴다는 점에서 의미가 있으며, 활성화 함수를 쓰는 일반 은닉층(비선형층)과 구분하는 표현일 뿐이다.

시그모이드(Sigmoid)와 기울기 소실

수식은 \(\sigma(x)=\dfrac{1}{1+e^{-x}}\). 출력이 0 또는 1에 가까워지면 그래프 기울기가 완만해진다. 이 구간에서 미분값은 0에 가까워지고, 역전파에서 0에 가까운 기울기가 계속 곱해지면 앞단(입력층 방향)으로 갈수록 기울기가 거의 전달되지 않는다. 이를 기울기 소실(Vanishing Gradient)이라 한다.

은닉층에서 시그모이드 지양

시그모이드는 또한 출력이 항상 양수라 원점 중심이 아니다(Not zero-centered). 이로 인한 편향 이동(bias shift)으로 층을 지날수록 분산이 커져 기울기 소실을 악화시킨다. 결론적으로 은닉층 활성화 함수로 시그모이드는 지양된다.

하이퍼볼릭탄젠트(tanh)

입력을 \(-1\)과 \(1\) 사이로 변환한다. 시그모이드와 같은 포화 문제는 있지만 원점 중심(zero-centered)이라 반환값 변화폭이 커, 시그모이드보다 기울기 소실이 덜하다. 그래서 은닉층에서 시그모이드보다 많이 쓰인다.

ReLU와 죽은 ReLU

수식은 \(f(x)=\max(0,x)\)로 매우 간단하다. 음수면 0, 양수면 입력 그대로 반환한다. 특정 양수값에 수렴하지 않아 깊은 신경망에서 시그모이드보다 훨씬 잘 작동하고, 단순 임계값이라 연산 속도도 빠르다. 단점은 입력이 음수면 기울기가 0이 되어 그 뉴런이 회생하기 어려운 죽은 ReLU(dying ReLU) 문제다.

Leaky ReLU

죽은 ReLU를 보완한다. 수식은 \(f(x)=\max(ax,x)\). 음수 입력에서도 0이 아닌 작은 기울기 \(a\)(보통 0.01)를 주어 뉴런이 죽지 않게 한다.

소프트맥스(Softmax)

\(\dfrac{e^{x_i}}{\sum_j e^{x_j}}\). 시그모이드처럼 주로 출력층에서 쓰이며, 시그모이드가 이진 분류용이라면 소프트맥스는 상호 배타적인 다중 클래스 분류용이다.

출력층 활성화 함수와 손실 함수의 관계
문제출력층 활성화 함수비용 함수
이진 분류시그모이드nn.BCELoss()
다중 클래스 분류소프트맥스nn.CrossEntropyLoss()
회귀없음MSE
자주 하는 실수

nn.CrossEntropyLoss()는 소프트맥스를 이미 포함한다. 따라서 출력층 로짓에 소프트맥스를 한 번 더 적용한 뒤 이 손실에 넘기면 안 된다. 스탠포드 cs231n도 ReLU를 먼저 쓰고, 그다음 Leaky ReLU·ELU를 시도하며, sigmoid는 쓰지 말라고 권장한다.

06·02

퍼셉트론(Perceptron)

퍼셉트론은 프랑크 로젠블라트가 1957년 제안한 초기 인공 신경망으로, 다수의 입력에서 하나의 결과를 내보내는 알고리즘이다. 뉴런이 일정 크기 이상의 신호를 받으면 축삭돌기로 신호를 전달하는 원리를 본떴다.

1. 퍼셉트론의 동작

입력값 \(x\)는 각각의 가중치 \(W\)와 곱해져 인공 뉴런으로 전달된다. 가중치가 클수록 그 입력이 중요하다는 뜻이다. 곱의 전체 합이 임계치(threshold, \(\theta\))를 넘으면 1, 아니면 0을 출력하며 이를 계단 함수(Step function)라 한다.

$$\text{if } \sum_i^n W_i x_i \ge \theta \Rightarrow y=1, \qquad \text{if } \sum_i^n W_i x_i < \theta \Rightarrow y=0$$

임계치를 좌변으로 넘기면 편향 \(b\)로 표현할 수 있다. 편향은 입력값이 1로 고정된 채 곱해지는 변수로, 가중치처럼 딥러닝이 학습해야 할 값 중 하나다.

$$\text{if } \sum_i^n W_i x_i + b \ge 0 \Rightarrow y=1, \qquad \text{if } \sum_i^n W_i x_i + b < 0 \Rightarrow y=0$$
로지스틱 회귀를 퍼셉트론보다 먼저 배운 이유가 여기 있다. 퍼셉트론의 활성화 함수를 계단 함수에서 시그모이드로 바꾸면 곧 이진 분류를 수행하는 로지스틱 회귀와 동일해진다. 즉 로지스틱 회귀 모델은 하나의 인공 뉴런으로 볼 수 있고, 둘의 차이는 오직 활성화 함수다.

2. 단층 퍼셉트론(Single-Layer Perceptron)

값을 보내는 입력층과 값을 받아 출력하는 출력층, 두 층으로만 이루어진다. 단층 퍼셉트론으로 AND·NAND·OR 게이트는 쉽게 구현할 수 있다. 예컨대 AND는 \([w_1,w_2,b]=[0.5,0.5,-0.7]\) 등으로 만족한다.

gates.py — AND / NAND / OR
def AND_gate(x1, x2):
    w1, w2, b = 0.5, 0.5, -0.7
    result = x1*w1 + x2*w2 + b
    return 0 if result <= 0 else 1

def NAND_gate(x1, x2):           # AND의 가중치·편향에 부호만 반전
    w1, w2, b = -0.5, -0.5, 0.7
    result = x1*w1 + x2*w2 + b
    return 0 if result <= 0 else 1

def OR_gate(x1, x2):
    w1, w2, b = 0.6, 0.6, -0.5
    result = x1*w1 + x2*w2 + b
    return 0 if result <= 0 else 1

단층 퍼셉트론의 한계 — XOR

입력이 서로 다를 때만 1을 내는 XOR 게이트는 단층 퍼셉트론으로 구현할 수 없다. 단층 퍼셉트론은 직선 하나로 두 영역을 나눌 수 있는(선형 분리 가능한) 문제만 풀 수 있기 때문이다. AND·NAND·OR은 직선으로 0과 1을 나눌 수 있지만, XOR은 불가능하다. (곡선, 즉 비선형 영역으로는 분리가 가능하다.)

3. 다층 퍼셉트론(MLP)

XOR은 AND·NAND·OR을 조합하면 만들 수 있다. 퍼셉트론 관점에서는 층을 더 쌓으면 된다. 입력층과 출력층 사이에 추가된 층을 은닉층(hidden layer)이라 하고, 은닉층을 가진 퍼셉트론을 다층 퍼셉트론(MLP)이라 한다.

  • 은닉층이 2개 이상인 신경망을 심층 신경망(Deep Neural Network, DNN)이라 한다.
  • 지금까지는 가중치를 수동으로 찾았지만, 이제는 손실 함수와 옵티마이저로 기계가 스스로 찾게 한다 — 이것이 학습(training)이다.
  • 심층 신경망을 학습시키는 것을 딥러닝(Deep Learning)이라 한다.
06·03

XOR 문제 구현하기

앞 장의 이론을 파이토치로 확인한다. 단층 퍼셉트론은 XOR에서 실패하고, 다층 퍼셉트론은 성공함을 코드로 보인다. 0/1 이진 분류이므로 비용 함수는 nn.BCELoss()를 쓴다.

1. 단층 퍼셉트론으로 XOR 풀기 (실패)

xor_single.py
import torch
import torch.nn as nn

device = 'cuda' if torch.cuda.is_available() else 'cpu'
torch.manual_seed(777)

X = torch.FloatTensor([[0,0],[0,1],[1,0],[1,1]]).to(device)
Y = torch.FloatTensor([[0],[1],[1],[0]]).to(device)

linear  = nn.Linear(2, 1, bias=True)
sigmoid = nn.Sigmoid()
model   = nn.Sequential(linear, sigmoid).to(device)

criterion = torch.nn.BCELoss().to(device)
optimizer = torch.optim.SGD(model.parameters(), lr=1)

for step in range(10001):
    optimizer.zero_grad()
    hypothesis = model(X)
    cost = criterion(hypothesis, Y)
    cost.backward()
    optimizer.step()
    if step % 100 == 0:
        print(step, cost.item())

2. 다층 퍼셉트론으로 XOR 풀기 (성공)

은닉층 3개를 쌓은 신경망이다. 입력층(2) → 은닉층1·2·3(각 10) → 출력층(1) 구조다.

xor_mlp.py
model = nn.Sequential(
    nn.Linear(2, 10, bias=True),   # 입력층 2 → 은닉층1 10
    nn.Sigmoid(),
    nn.Linear(10, 10, bias=True),  # 은닉층1 10 → 은닉층2 10
    nn.Sigmoid(),
    nn.Linear(10, 10, bias=True),  # 은닉층2 10 → 은닉층3 10
    nn.Sigmoid(),
    nn.Linear(10, 1, bias=True),   # 은닉층3 10 → 출력층 1
    nn.Sigmoid()
).to(device)

criterion = torch.nn.BCELoss().to(device)
optimizer = torch.optim.SGD(model.parameters(), lr=1)

for epoch in range(10001):
    optimizer.zero_grad()
    hypothesis = model(X)
    cost = criterion(hypothesis, Y)
    cost.backward()
    optimizer.step()
    if epoch % 100 == 0:
        print(epoch, cost.item())
출력 — 비용이 0에 수렴, 정확도 100%

평가 시 no_grad

평가 단계는 학습이 아니므로 with torch.no_grad(): 블록에서 기울기 계산을 끄면 연산이 빨라지고 메모리를 아낀다. (hypothesis > 0.5).float()로 0.5를 기준 삼아 이진 레이블을 만든다.

06·04

역전파(BackPropagation)

순전파로 오차를 구한 뒤, 경사 하강법으로 가중치를 어떻게 업데이트하는지 직접 손계산으로 이해한다. 입력 2 · 은닉 2 · 출력 2의 신경망이며, 은닉·출력층 모두 시그모이드를 쓰고 편향은 고려하지 않는다. 학습률 \(\alpha=0.5\).

1. 순전파(Forward Propagation)

가중합 \(z\)는 활성화 함수 이전 값, \(h\)·\(o\)는 시그모이드 통과 후 값이다. (초기값: \(W_1{=}.3, W_2{=}.25, W_3{=}.4, W_4{=}.35, W_5{=}.45, W_6{=}.4, W_7{=}.7, W_8{=}.6\), 입력 \(x_1{=}.1, x_2{=}.2\), 실제값 \(.4, .6\))

$$z_1 = W_1x_1 + W_2x_2 = 0.08, \qquad z_2 = W_3x_1 + W_4x_2 = 0.11$$
$$h_1 = \sigma(z_1) = 0.51998934, \qquad h_2 = \sigma(z_2) = 0.52747230$$
$$z_3 = W_5h_1 + W_6h_2 = 0.44498412, \qquad z_4 = W_7h_1 + W_8h_2 = 0.68047592$$
$$o_1 = \sigma(z_3) = 0.60944600, \qquad o_2 = \sigma(z_4) = 0.66384491$$

손실 함수로 MSE를 쓰고 각 오차를 더해 전체 오차 \(E_{total}\)을 구한다.

$$E_{o1}=\tfrac12(target_{o1}-output_{o1})^2=0.02193381,\quad E_{o2}=0.00203809$$
$$E_{total}=E_{o1}+E_{o2}=0.02397190$$

2. 역전파 1단계 — 출력층과 가까운 가중치

출력층과 은닉층 사이의 \(W_5,W_6,W_7,W_8\)을 업데이트한다. \(W_5\)를 예로, 연쇄 법칙으로 \(\dfrac{\partial E_{total}}{\partial W_5}\)을 분해한다.

$$\frac{\partial E_{total}}{\partial W_5} = \frac{\partial E_{total}}{\partial o_1}\times\frac{\partial o_1}{\partial z_3}\times\frac{\partial z_3}{\partial W_5}$$

세 항을 순서대로 계산한다. (시그모이드 미분은 \(f(x)(1-f(x))\)임을 기억한다.)

$$\frac{\partial E_{total}}{\partial o_1}=-(target_{o1}-output_{o1})=0.20944600$$
$$\frac{\partial o_1}{\partial z_3}=o_1(1-o_1)=0.23802157, \qquad \frac{\partial z_3}{\partial W_5}=h_1=0.51998934$$
$$\frac{\partial E_{total}}{\partial W_5}=0.20944600\times0.23802157\times0.51998934=0.02592286$$

경사 하강법으로 업데이트한다.

$$W_5^{+}=W_5-\alpha\frac{\partial E_{total}}{\partial W_5}=0.45-0.5\times0.02592286=0.43703857$$

같은 원리로 \(W_6^{+}=0.38685205,\ W_7^{+}=0.69629578,\ W_8^{+}=0.59624247\).

3. 역전파 2단계 — 입력층과 가까운 가중치

이제 \(W_1,W_2,W_3,W_4\)를 업데이트한다. \(W_1\)의 첫 항 \(\dfrac{\partial E_{total}}{\partial h_1}\)은 두 출력 경로의 오차가 모두 \(h_1\)을 거치므로 둘을 더해 구한다.

$$\frac{\partial E_{total}}{\partial h_1}=\frac{\partial E_{o1}}{\partial h_1}+\frac{\partial E_{o2}}{\partial h_1}=0.02243370+0.00997311=0.03240681$$
$$\frac{\partial h_1}{\partial z_1}=h_1(1-h_1)=0.24960043, \qquad \frac{\partial z_1}{\partial W_1}=x_1=0.1$$
$$\frac{\partial E_{total}}{\partial W_1}=0.03240681\times0.24960043\times0.1=0.00080888$$
$$W_1^{+}=0.1-0.5\times0.00080888=0.29959556$$

4. 결과 확인

업데이트된 가중치로 다시 순전파하면 전체 오차가 줄어든다.

단 1회 역전파의 효과

\(E_{total}\)이 0.02397190 → 0.02323634로 감소했다. 인공 신경망의 학습은 이렇게 오차를 최소화하는 가중치를 찾기 위해 순전파와 역전파를 반복하는 과정이다.

06·05

다층 퍼셉트론으로 손글씨 분류하기

사이킷런이 제공하는 load_digits() 데이터로 MLP 분류기를 만든다. 각 이미지는 0~15 명암의 \(8\times8=64\)픽셀 흑백 이미지이며, 총 1,797개다. (MNIST와는 다른 데이터다.)

1. 데이터 소개

digits.images는 각 샘플을 \(8\times8\) 행렬로, digits.data는 이를 64차원 벡터로 펼쳐 저장한다. 신경망 입력에는 digits.data(64차원 벡터)를 쓰는 것이 낫다.

2. MLP 분류기 만들기

입력 64 → 은닉 32 → 은닉 16 → 출력 10(클래스 수) 구조다. 다중 클래스 분류이므로 손실은 소프트맥스를 포함한 CrossEntropyLoss를 쓴다.

digits_mlp.py
import torch
import torch.nn as nn
from torch import optim
from sklearn.datasets import load_digits

digits = load_digits()
X = digits.data        # (1797, 64) 특성 행렬
Y = digits.target      # 각 이미지의 레이블

model = nn.Sequential(
    nn.Linear(64, 32),   # 입력층 64 → 은닉층 32
    nn.ReLU(),
    nn.Linear(32, 16),   # 은닉층 32 → 은닉층 16
    nn.ReLU(),
    nn.Linear(16, 10)    # 은닉층 16 → 출력층 10 (클래스 수)
)

X = torch.tensor(X, dtype=torch.float32)
Y = torch.tensor(Y, dtype=torch.int64)

loss_fn   = nn.CrossEntropyLoss()   # 소프트맥스를 이미 포함
optimizer = optim.Adam(model.parameters())

losses = []
for epoch in range(100):
    optimizer.zero_grad()
    y_pred = model(X)               # 순전파
    loss   = loss_fn(y_pred, Y)     # 비용 계산
    loss.backward()                 # 역전파로 기울기 계산
    optimizer.step()                # 파라미터 업데이트
    if epoch % 10 == 0:
        print('Epoch {:4d}/{} Cost: {:.6f}'.format(epoch, 100, loss.item()))
    losses.append(loss.item())
출력 — 비용이 꾸준히 감소
Epoch    0/100 Cost: 2.380815
Epoch   10/100 Cost: 2.059323
...
Epoch   90/100 Cost: 0.205398
학습 루프의 4단계

zero_grad() 기울기 초기화 → model(X) 순전파 예측 → loss.backward() 기울기 계산 → optimizer.step() 파라미터 업데이트. 이 네 줄이 모든 학습 루프의 뼈대다.

06·06

다층 퍼셉트론으로 MNIST 분류하기

소프트맥스 회귀(단층)로 풀던 MNIST를, 이번엔 은닉층을 추가한 다층 퍼셉트론으로 분류한다. 입력 784(28×28) → 은닉 100 → 은닉 100 → 출력 10 구조다.

1. 데이터 로드와 전처리

fetch_openml로 불러온 뒤, 레이블을 정수형으로 바꾸고 모든 픽셀을 0–255에서 0–1로 정규화한다. 정규화는 학습 효율을 높인다.

mnist_load.py
import numpy as np
from sklearn.datasets import fetch_openml

mnist = fetch_openml('mnist_784', version=1, cache=True, as_frame=False)
mnist.target = mnist.target.astype(np.int8)

X = mnist.data / 255    # 0-255 → [0,1] 정규화
y = mnist.target

2. 훈련/테스트 분리와 DataLoader

텐서로 변환 후 TensorDataset으로 묶고 DataLoader에 넣으면 미니배치 학습이 편해진다.

mnist_loader.py
import torch
from torch.utils.data import TensorDataset, DataLoader
from sklearn.model_selection import train_test_split

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=1/7, random_state=0)

X_train = torch.Tensor(X_train); X_test = torch.Tensor(X_test)
y_train = torch.LongTensor(y_train); y_test = torch.LongTensor(y_test)

ds_train = TensorDataset(X_train, y_train)
ds_test  = TensorDataset(X_test,  y_test)

loader_train = DataLoader(ds_train, batch_size=64, shuffle=True)
loader_test  = DataLoader(ds_test,  batch_size=64, shuffle=False)

3. 모델 정의와 학습

mnist_train.py
from torch import nn, optim

model = nn.Sequential()
model.add_module('fc1',   nn.Linear(28*28*1, 100))
model.add_module('relu1', nn.ReLU())
model.add_module('fc2',   nn.Linear(100, 100))
model.add_module('relu2', nn.ReLU())
model.add_module('fc3',   nn.Linear(100, 10))

loss_fn   = nn.CrossEntropyLoss()
optimizer = optim.Adam(model.parameters(), lr=0.01)

epochs = 3
for epoch in range(epochs):
    for data, targets in loader_train:
        optimizer.zero_grad()
        y_pred = model(data)
        loss = loss_fn(y_pred, targets)
        loss.backward()
        optimizer.step()
    print('Epoch {:4d}/{} Cost: {:.6f}'.format(epoch+1, 3, loss.item()))

4. 평가

model.eval()로 추론 모드 전환 후 torch.no_grad() 안에서 평가한다. torch.max(outputs.data, 1)은 각 예측에서 가장 확률 높은 클래스를 찾아준다.

mnist_eval.py
model.eval()                       # 추론 모드 (학습/추론 시 다르게 동작하는 계층 대비)
correct = 0
with torch.no_grad():              # 추론에는 미분 불필요
    for data, targets in loader_test:
        outputs = model(data)
        _, predicted = torch.max(outputs.data, 1)   # 최고 확률 레이블
        correct += predicted.eq(targets.data.view_as(predicted)).sum()

data_num = len(loader_test.dataset)
print('정확도: {}/{} ({:.0f}%)'.format(correct, data_num, 100.*correct/data_num))

테스트 데이터에서 예측 정확도: 9566/10000 (96%)

06·07

과적합(Overfitting)을 막는 방법들

과적합은 훈련 데이터를 불필요할 정도로 암기해 노이즈까지 학습한 상태로, 검증·테스트 데이터에서 성능을 떨어뜨린다. 신경망의 과적합을 막는 네 가지 방법을 정리한다.

1. 데이터의 양 늘리기

데이터가 적으면 특정 패턴·노이즈까지 쉽게 암기한다. 데이터를 늘리면 일반적인 패턴을 학습해 과적합을 방지한다. 양이 부족하면 기존 데이터를 조금씩 변형·추가하는 데이터 증강(Data Augmentation)을 쓴다. 이미지의 경우 회전·노이즈 추가·일부 수정 등이 흔하다.

2. 모델의 복잡도 줄이기

신경망의 복잡도는 은닉층 수나 매개변수 수로 결정된다. 과적합이 보이면 층이나 매개변수를 줄인다. 매개변수 수를 모델의 수용력(capacity)이라 부르기도 한다.

reduce_capacity.py — 3층을 2층으로
class Architecture1(nn.Module):
    def __init__(self, input_size, hidden_size, num_classes):
        super().__init__()
        self.fc1  = nn.Linear(input_size, hidden_size)
        self.relu = nn.ReLU()
        self.fc2  = nn.Linear(hidden_size, num_classes)   # 은닉층 하나 제거

    def forward(self, x):
        out = self.fc1(x)
        out = self.relu(out)
        out = self.fc2(out)
        return out

3. 가중치 규제(Regularization)

복잡한 모델일수록 과적합 가능성이 높다. 비용 함수에 가중치 항을 더해 가중치를 작게 유지하도록 강제한다.

$$\text{L1: } \text{cost} + \lambda\sum_w |w| \qquad\qquad \text{L2: } \text{cost} + \tfrac12\lambda\sum_w w^2$$
  • L1 규제 — 가중치 절대값 합을 추가. \(w\)가 0 또는 0에 가까워져 일부 특성이 거의 안 쓰이게 된다. 어떤 특성이 모델에 영향을 주는지 판단할 때 유용하다.
  • L2 규제 — 가중치 제곱합을 추가. \(w\)가 완전히 0이 되기보다 0에 가까워진다. 특별한 판단이 필요 없다면 경험적으로 더 잘 동작하므로 L2를 권장한다.

\(\lambda\)는 규제 강도를 정하는 하이퍼파라미터다. 신경망에서 L2 규제는 가중치 감쇠(weight decay)라고도 부르며, 파이토치에서는 옵티마이저의 weight_decay 인자로 적용한다(기본값 0).

weight_decay.py
model = Architecture1(10, 20, 2)
optimizer = torch.optim.Adam(model.parameters(), lr=1e-4, weight_decay=1e-5)
용어 혼동 주의

Regularization을 '정규화'로 옮기기도 하지만 Normalization(정규화)과 혼동되므로 규제 또는 정형화로 부르는 것이 바람직하다. 신경망에서 Normalization 계열로는 배치 정규화·층 정규화가 있다(다음 장).

4. 드롭아웃(Dropout)

학습 과정에서 신경망의 일부를 사용하지 않는 방법이다. 비율을 0.5로 하면 학습마다 무작위로 절반의 뉴런만 사용한다.

드롭아웃은 학습 시에만 쓰고 예측 시에는 쓰지 않는 것이 일반적이다. 특정 뉴런·조합에 과하게 의존하는 것을 막고, 매번 다른 뉴런 조합을 쓰므로 서로 다른 신경망을 앙상블하는 효과를 내어 과적합을 방지한다.
06·08

기울기 소실(Vanishing)과 폭주(Exploding)

깊은 신경망에서 역전파 시 입력층으로 갈수록 기울기가 작아져 가중치가 갱신되지 않는 것이 기울기 소실, 반대로 기울기가 비정상적으로 커져 발산하는 것이 기울기 폭주다(주로 RNN에서 발생). 이를 막는 방법들을 다룬다.

1. ReLU와 ReLU의 변형들

시그모이드는 입력 절대값이 크면 출력이 0/1에 수렴해 기울기가 0에 가까워지고, 역전파에서 전파할 기울기가 사라진다.

은닉층 활성화 함수 원칙

은닉층에서 시그모이드·tanh 대신 ReLU나 Leaky ReLU를 쓴다. Leaky ReLU는 모든 입력에서 기울기가 0에 수렴하지 않아 죽은 ReLU 문제까지 해결한다. 한 마디로: 은닉층에서는 시그모이드를 쓰지 말 것.

2. 가중치 초기화(Weight Initialization)

같은 모델이라도 초기 가중치에 따라 결과가 달라진다. 초기화만 적절히 해도 기울기 소실을 완화할 수 있다. 이전 층 뉴런 수 \(n_{in}\), 다음 층 뉴런 수 \(n_{out}\)로 식을 세운다.

세이비어 초기화(Xavier / Glorot, 2010)

균등 분포와 정규 분포 두 경우로 나뉘며, 이전 층과 다음 층 뉴런 수를 모두 반영한다.

$$W \sim Uniform\!\left(-\sqrt{\frac{6}{n_{in}+n_{out}}},\ +\sqrt{\frac{6}{n_{in}+n_{out}}}\right) \qquad \sigma=\sqrt{\frac{2}{n_{in}+n_{out}}}$$

여러 층의 기울기 분산 사이에 균형을 맞춰준다. 단, S자형(시그모이드·tanh)에는 잘 맞지만 ReLU와는 성능이 좋지 않다.

He 초기화(He, 2015)

세이비어와 유사하나 다음 층 뉴런 수를 반영하지 않고 \(n_{in}\)만 사용한다.

$$W \sim Uniform\!\left(-\sqrt{\frac{6}{n_{in}}},\ +\sqrt{\frac{6}{n_{in}}}\right) \qquad \sigma=\sqrt{\frac{2}{n_{in}}}$$
선택 기준

시그모이드·tanh → 세이비어 초기화가 효율적. ReLU 계열 → He 초기화가 효율적. 실무에서는 ReLU + He 초기화 조합이 더 보편적이다.

3. 배치 정규화(Batch Normalization)

ReLU와 He 초기화로도 훈련 중 기울기 소실/폭주가 다시 생길 수 있다. 배치 정규화는 각 층 입력을 평균·분산으로 정규화해 학습을 안정시킨다.

내부 공변량 변화(Internal Covariate Shift)
  • 공변량 변화 — 훈련 데이터 분포와 테스트 데이터 분포가 다른 경우.
  • 내부 공변량 변화 — 학습 중 층 사이에서 입력 데이터 분포가 달라지는 현상. 이전 층의 가중치가 바뀌면 현재 층에 오는 입력 분포가 달라진다.
수식

배치 단위로 정규화하며, 활성화 함수를 통과하기 전에 수행한다. 정규화 후 스케일 \(\gamma\)와 시프트 \(\beta\)(둘 다 학습 대상)를 적용한다.

$$\mu_B \leftarrow \frac{1}{m}\sum_{i=1}^{m}x^{(i)} \qquad \sigma_B^2 \leftarrow \frac{1}{m}\sum_{i=1}^{m}(x^{(i)}-\mu_B)^2$$
$$\hat{x}^{(i)} \leftarrow \frac{x^{(i)}-\mu_B}{\sqrt{\sigma_B^2+\varepsilon}} \qquad y^{(i)} \leftarrow \gamma\,\hat{x}^{(i)}+\beta = BN_{\gamma,\beta}(x^{(i)})$$

\(\varepsilon\)은 분모가 0이 되는 것을 막는 작은 수(보통 \(10^{-5}\)). 학습 시 배치별 평균·분산으로 이동 평균/분산을 저장해두었다가, 테스트 시엔 저장해둔 값으로 정규화한다.

배치 정규화의 장점

시그모이드·tanh를 써도 기울기 소실이 크게 개선되고, 가중치 초기화에 훨씬 덜 민감해진다. 더 큰 학습률을 쓸 수 있어 학습 속도가 빨라지고, 미니배치마다 잡음을 넣는 부수 효과로 과적합도 일부 방지한다(드롭아웃과 함께 쓰는 게 좋다).

배치 정규화의 한계

① 미니배치 크기에 의존적 — 배치가 너무 작으면(특히 1이면 분산이 0) 잘 동작하지 않는다. ② RNN에 적용하기 어렵다 — RNN은 시점마다 통계치가 달라 적용이 까다롭다. 그 대안이 층 정규화다.

4. 층 정규화(Layer Normalization)

배치 정규화는 같은 특성끼리(열 방향) 여러 샘플에 걸쳐 평균·분산을 구한다. 반면 층 정규화는 각 샘플 안에서(행 방향) 모든 특성에 걸쳐 평균·분산을 구한다. 그래서 배치 크기에 의존하지 않고 RNN에도 적용하기 수월하다.

구분정규화 방향배치 크기 의존RNN 적용
배치 정규화특성별(열) · 샘플들에 걸쳐의존적어려움
층 정규화샘플별(행) · 특성들에 걸쳐독립적수월함
다음 장은 07. 순환 신경망(RNN)이다. RNN은 시점마다 통계가 다르고 기울기 폭주가 일어날 수 있어, 여기서 배운 층 정규화가 자연스럽게 이어진다.

활성화 함수 비교

은닉층은 ReLU 계열, 출력층은 문제에 맞춰 선택한다.

함수수식주 용도특징 / 문제점
Sigmoid\(\frac{1}{1+e^{-x}}\)출력층(이진)기울기 소실, not zero-centered. 은닉층 지양
tanh\(\tanh(x)\)은닉층zero-centered, 시그모이드보다 소실 덜함

문제출력층 활성화손실 함수

과적합을 막는 4가지 방법

  1. 데이터 양 늘리기 — 부족하면 데이터 증강(Augmentation).
  2. 모델 복잡도 줄이기 — 은닉층·매개변수(수용력) 축소.
  3. 가중치 규제 — L1(절대값 합) / L2(제곱합, =weight decay, 권장).
  4. 드롭아웃 — 학습 시에만 일부 뉴런 무작위 제외, 앙상블 효과.

기울기 소실 / 폭주 해결법

  1. ReLU 계열 활성화 함수 — 은닉층에 ReLU·Leaky ReLU.
  2. 가중치 초기화 — 시그모이드/tanh엔 Xavier, ReLU엔 He.
  3. 배치 정규화 — 활성화 함수 전, 배치 단위 정규화(+\(\gamma,\beta\)).
  4. 층 정규화 — 샘플 내 정규화, 배치 크기 무관·RNN 친화.

가중치 초기화 공식 (정규 분포 기준)

균등 분포는 분모를 그대로 두고 \(\sqrt{6/\cdot}\) 범위의 \(Uniform\)을 쓴다.

방법표준편차 \(\sigma\)반영 대상적합 활성화
Xavier\(\sqrt{\dfrac{2}{n_{in}+n_{out}}}\)\(n_{in}\)·\(n_{out}\) 모두시그모이드 · tanh
He\(\sqrt{\dfrac{2}{n_{in}}}\)\(n_{in}\)만ReLU 계열

핵심 한 줄 정리

  • 단층 퍼셉트론 = 선형 분리만 가능 → XOR 불가. 은닉층을 쌓은 MLP가 해결.
  • 학습 루프 4단계 = zero_grad → forward → backward → step.
  • 은닉층 활성화는 ReLU, 출력층 + 손실은 문제 유형에 맞게.
  • CrossEntropyLoss에 소프트맥스를 중복 적용 금지.
  • 깊은 망의 두 적 = 과적합(규제·드롭아웃)과 기울기 소실(ReLU·초기화·정규화).
